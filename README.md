# 학교 행정업무 혁신 (SAI-501) — AI-Native 코스 패키지

인공지능을 활용해 학교 행정업무를 혁신하는 프로젝트 기반 대학원 수업(수강생: 현직교사)의 코스 패키지 저장소입니다. 하이브리드 AI-Native LMS PoC의 샘플 강의로, 실라버스와 자원이 사람 사용자용과 에이전트(AI)용으로 명시적으로 분리되어 있습니다.

## 두 개의 진입점

| 사용자 | 시작 문서 | 내용 |
| --- | --- | --- |
| 사람 (수강생·교수자) | syllabus/SYLLABUS_HUMAN.md | 15주 상세 강의계획서 — 주차별 질문·수업 흐름·과제·AI 비서 활용 |
| 에이전트 (로컬/서버 AI) | syllabus/syllabus_agent.json | 기계가독 실라버스 — 모듈·활동·민감도·라우팅 힌트·자원 바인딩 |

에이전트는 agent_resources/interface.json에서 시작해 자원 매니페스트(22항목)와 코스 스킬 5종(agent_resources/skills/)으로 연결됩니다.

## 구성

- syllabus/ — 사람용 실라버스(md) + 에이전트용 실라버스(json)
- contents/M1/ — 완성 샘플 모듈: 읽기자료 2, 실습 안내, 과제 명세, 비식별 데이터셋 2종
- contents/M2..M6/ — 모듈 구조 스텁 (자원 매핑 완료, 본문은 후속 라운드)
- agent_resources/ — 인터페이스, 자원 매니페스트, 코스 스킬 5종
- architecture/ — 코스 지식베이스 허브 아키텍처 (단일 출처 → 4중 파생)
- _spec/ — 공유 작성 스펙 (후속 콘텐츠 라운드 재사용)
- PROVENANCE.md — 원천 자원·비식별 처리 기록

## 코스 개요

- 6모듈 × 15주. ① 어떤 행정업무가 발생하는가(데이터 분석) ② 왜 발생하는가(L0–L4 조건 레이어) ③ 왜 그가 담당하는가(배분·검토경계) ④ 데이터 파이프라인 설계 ⑤ AI 활용 전략(시뮬레이터 실습) ⑥ AI 활용 혁신 계획 수립.
- 근거. 공개 행정문서 20,755건 수집·분석 연구 자산에서 파생 — 대표학교 연간 업무사건 714건, 업무 도메인 10개, 일반모델 조건 단위 320개(L0–L4).
- 프라이버시 원칙. 수강생의 자기 학교 업무 기록 등 민감 활동은 에이전트 실라버스에 sensitivity: sensitive + routing_hint: local로 표기 — 로컬 AI(기기 내)에서만 처리.
