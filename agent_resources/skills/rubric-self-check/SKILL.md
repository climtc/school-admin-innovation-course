# rubric-self-check — 루브릭 자가 점검

## 목적
학생 산출물을 해당 모듈의 rubric_ref 기준(교사 검증 카드 형식, 6차원 루브릭, 허용 자동화 수준 판정 기준)에 비추어 점검하고, 기준별 충족/미흡과 구체적 개선 제안을 돌려준다. 공식 채점이 아니라 제출 전 자가 개선 도구다.

## 입력
- 학생 산출물 원문 (과제 텍스트, 분석 메모, 계획서 초안 등)
- 대상 activity_id 또는 module_id — rubric_ref 결정에 사용
- (선택) 동료평가 모드 여부 (M4-A4, M6-A3)

## 절차
1. 학생이 자기 과제 원문을 입력하는 순간 sensitivity=sensitive, routing=local을 강제 적용하고, 원문이 로컬에서만 처리됨을 고지한다.
2. `syllabus/syllabus_agent.json`에서 해당 모듈의 assessment.rubric_ref를 확인한다 (M1–M4: kb://validation/teacher_cards, M5–M6: kb://validation/ai_strategy_packet).
3. rubric_ref 자원에서 점검 기준을 추출한다: 교사 검증 카드는 현실 부합성·실행 가능성·위험 통제, AI 전략 패킷은 허용 자동화 수준 5단계 판정의 적절성, 시뮬레이터 연계 산출물은 6차원 루브릭.
4. 기준별로 산출물을 대조하여 충족 / 부분 충족 / 미흡 + 해당 근거 구절을 표시한다.
5. 코스 공통 필수 점검 3종을 항상 추가한다: ①비식별(학교명·인명·지역명 잔존 여부 — "대표학교" 외 고유명 검출) ②근거 인용(사실 주장마다 resource_id 또는 §3 고정 수치 인용 여부) ③산출물 요건(assessment.artifact 구성 요소 누락 여부).
6. 미흡 항목마다 "무엇을 어떻게 고칠지" 1~2문장의 실행 가능한 개선 제안을 붙인다. 단, 수정 문장을 대신 써 주지는 않는다.
7. 동료평가 모드면 점검표를 동료평가용 질문 목록으로 변환해 출력하고, 동료 산출물에 대해서는 비식별본만 다루도록 안내한다.

## 사용하는 resource_id
- `kb://validation/teacher_cards` — M1~M4 산출물 점검 기준 (TVC 카드 형식)
- `kb://validation/ai_strategy_packet` — M5~M6 점검 기준 (AIC-0001~0030, 허용 자동화 수준 5단계)
- `kb://simulator/app` — 6차원 루브릭 정의 (M5 시뮬레이터 연계 산출물)
- `kb://whitepaper/stage1` ~ `kb://whitepaper/stage9` — M6 계획서 구조 완결성 대조

## 라우팅·민감도 규칙
- 학생이 자기 과제 원문을 다루는 모든 점검: sensitivity=sensitive, routing=local 강제. 원문·점검 결과 모두 서버 전송 금지.
- 동료평가 모드에서 동료 산출물을 받을 때는 비식별본인지 먼저 확인하고, 비식별이 확인되지 않으면 점검을 거부하고 작성자에게 비식별 처리를 요청하도록 안내한다.
- 루브릭 기준 자체에 대한 일반 질문(산출물 미포함)은 either 허용.

## 출력 형식
```
[점검 기준] rubric_ref + 기준 목록
[기준별 판정] 충족/부분/미흡 + 근거 구절
[공통 점검] 비식별 / 근거 인용 / 요건 충족
[개선 제안] 미흡 항목별 1~2문장
[주의] 본 점검은 공식 평가를 대체하지 않음
```

## 금지사항
- 점수·등급 부여 금지 — 충족/부분/미흡 판정과 개선 제안까지만.
- 미흡 항목의 수정 문장 대필 금지.
- 비식별 미확인 동료 산출물 점검 금지.
- rubric_ref 자원에 없는 기준을 임의 추가하여 판정하는 것 금지 (공통 점검 3종은 코스 정책으로 예외).
