# 과제 — 자기 학교 업무 AI 활용 전략 초안

> M5 · 과제 | 자원: `kb://validation/ai_strategy_packet` · `kb://simulator/app` · `kb://calendar/work_event_candidates_714`

## 과제 개요

12주차 평가 결과를 바탕으로, 소속 학교 업무에 대한 **AI 활용 전략 초안**을 작성한다. 어떤 업무를 어떤 자동화 수준으로 다룰지, 사람 검토 경계를 어디에 둘지, 민감 데이터를 어떻게 처리할지를 하나의 전략으로 묶는다. M6 최종 계획서의 핵심 입력이 된다.

## 제출 형식

### 1. 업무별 전략표

| 업무 | 도메인 | 권장 자동화 수준 | 검토 경계 | 데이터 처리 위치(로컬/서버) | 기대 효과 |
|---|---|---|---|---|---|
| (업무1) | | draft~not_allowed | | | |
| (5건) | | | | | |

### 2. 전략 서술 (3문항)

1. **우선 자동화 대상 1~2건.** draft/verify로 가장 큰 부담을 더는 업무와 그 근거.
2. **통제 강화 대상 1~2건.** assist_only/not_allowed로 두어야 하는 업무와 그 위험.
3. **데이터 주권 설계.** 민감 데이터(개인정보·평가·상담)를 기기 내 처리로 두는 설계와 그 이유.

## 평가 루브릭 (4기준 × 3수준)

| 기준 | 미흡(1) | 보통(2) | 우수(3) |
|---|---|---|---|
| 수준 판단 | 자동화 수준 근거 약함 | 수준 부여됨 | 업무별 수준이 위험·효과 근거로 정당화됨 |
| 검토 경계 | 누락·모호 | 경계 식별 | 경계 설정의 타당성을 데이터로 뒷받침 |
| 데이터 주권 | 미고려 | 언급 | 민감 데이터의 처리 위치를 명시·정당화 |
| 실행 가능성 | 일반론 | 일부 구체 | 자기 학교에서 바로 실행 가능한 수준 |

## 비식별 체크리스트

- [ ] 특정 학교·인명·지역 미사용.
- [ ] 업무 예시에 개인 식별 정보 없음.
- [ ] 민감 데이터 항목을 구체 인물과 연결하지 않음.
