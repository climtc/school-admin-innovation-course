# M1 — 행정업무의 실태: 무엇이 발생하는가 (1–3주)

> 코스: SAI-501 학교 행정업무 혁신 | 대상: 교육대학원 석사과정 현직교사 | course_package v0.1.0

## 핵심 질문

**학교에서 1년 동안 어떤 행정업무가, 언제, 얼마나 발생하는가 — 그리고 그것을 데이터로 어떻게 확인할 수 있는가?**

여러분은 행정업무를 매일 "겪고" 있지만, 그것을 "세어 본" 적은 드뭅니다. M1은 인상과 푸념이 아니라 공개 데이터로 학교 행정업무의 실태를 분석하는 모듈입니다. 대표학교의 연간 업무사건 후보 714건(2025.3–2026.2)을 직접 다루면서, 마지막에는 자기 소속 학교의 업무를 같은 방식으로 기록하는 데까지 나아갑니다.

## 1–3주 흐름

| 주차 | 주제 | 활동 | 콘텐츠 |
|---|---|---|---|
| 1주 | 공개 행정문서는 업무의 흔적이다 | 읽기·강의: 공개자료로 학교 업무를 재구성하는 방법론. 공개 행정문서 1차 수집 20,755건 → 분석 가능 파일 15,851건 → 업무사건 후보 714건이 만들어진 과정 | `reading_01_traces_of_work.md` |
| 2주 | 업무사건 데이터 첫 분석 | 읽기: 10개 도메인 지형. 실습: 도메인별 분포·월별 피크(3월 87건 → 2월 160건)·자기 직무 필터링 | `reading_02_domain_landscape.md`, `activity_week2_data_analysis.md` |
| 3주 | 내 학교의 업무 인벤토리 | 과제 수행·공유: 자기 학교 업무 5건을 업무사건 스키마로 기록, 데이터에 안 보이는 업무(가시성 결손) 토론 | `assignment_my_school_inventory.md` |

## 산출물

**나의 학교 업무 인벤토리** — 자기 소속 학교의 최근 업무 5건을 대표학교 데이터셋과 같은 스키마(도메인·트리거·담당·시스템·산출문서)로 기록한 표. 개인정보·학교 식별 정보를 제거한 비식별 버전으로 제출하며, M2(발생 원인 분석)와 M6(혁신 계획서)의 기초 자료가 됩니다.
