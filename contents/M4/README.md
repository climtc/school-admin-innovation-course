# M4 — 데이터 파이프라인 설계 (8–10주) [구조 스텁]

> 코스: SAI-501 | 상태: 콘텐츠 작성 예정 (course_package v0.1.0 기준 구조만 확정)

## 핵심 질문

행정업무 혁신을 위한 수집→전처리→분석 파이프라인을 어떻게 설계하는가? 공개 행정문서 1차 수집 20,755건이 분석 가능 파일 15,851건, 업무사건 후보 714건이 되기까지의 파이프라인을 해부하고, 자기 학교 맥락의 파이프라인을 설계한다.

## 주차

- 8주: 수집 단계 — 정보공개포털 수집 스크립트 구조와 검색 전략 분석
- 9주: 전처리·추출 단계 — 파일 전처리와 업무사건 추출 로직, 중복 제거와 추적성
- 10주: 설계 실습 — 자기 학교 업무 데이터 파이프라인 설계안 작성 (비식별·윤리 원칙 포함)
